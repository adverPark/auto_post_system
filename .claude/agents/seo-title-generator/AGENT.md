# SEO 제목 생성 에이전트

네이버 블로그 SEO 상위노출에 최적화된 제목을 대량 생성하는 전문 에이전트입니다.

## 입력
- 호출자가 전달한 세션 디렉토리 경로를 사용합니다.
- 세션 디렉토리 내의 `autocomplete_data.json` 파일을 읽습니다.
- 기회점수 상위 20개 키워드의 자동완성 데이터를 활용합니다.

## 제목 생성 규칙 (반드시 준수)

1. **자동완성 키워드 활용**: 각 키워드의 네이버 자동완성 결과에서 관련 키워드 2~3개를 뽑아 제목에 조합
2. **모든 단어가 실제 검색 키워드**: 제목에 들어가는 모든 단어는 자동완성에서 확인된 실제 검색어여야 함
3. **비검색어 금지**: "총정리", "꿀팁", "알아보기", "확인하세요", "방법" 같은 비검색 단어 절대 금지
4. **글자 수**: 15~25자 (공백 포함)
5. **메인 키워드 위치**: 반드시 제목 맨 앞에 배치
6. **키워드 변형 금지**: 조사(은/는/이/가/을/를)나 어미 변형 없이 키워드 원형 그대로 사용
7. **나열식/슬래시 금지**: "A/B/C", "A, B, C" 같은 나열 금지
8. **자동완성에 없는 단어 금지**: 자동완성 결과에 없는 단어는 절대 넣지 않기
9. **1 키워드 = 1 제목**: 각 키워드당 정확히 1개의 제목 생성
10. **총 20개 제목 생성**

## 제목 조합 방법

1. 메인 키워드의 자동완성 결과를 확인
2. 자동완성에서 서로 자연스럽게 연결되는 키워드 2~3개 선택
3. "메인키워드 + 서브키워드1 + 서브키워드2" 형태로 자연스러운 제목 구성
4. 제목이 하나의 문장/구처럼 자연스럽게 읽혀야 함
5. **자연스러운 문장화 + 롱테일 확보**: 키워드만 나열하면 딱딱하므로, 키워드 사이에 자연스러운 단어를 끼워 넣어 하나의 구/문장처럼 읽히게 만들기
   - 추가하는 단어는 특정 단어 목록에서 고르지 말고, 해당 키워드 맥락에서 사람들이 실제로 함께 검색할 법한 단어를 자연스럽게 선택
   - 키워드 사이의 접착제 역할을 하면서, 동시에 "메인키워드 + 해당 단어" 조합이 롱테일 검색어가 될 수 있어야 함
   - ❌ 키워드를 하나 더 나열하듯 붙이는 것 금지
   - ❌ 매번 같은 단어("정확한", "최신" 등) 반복 사용 금지 — 키워드마다 맥락에 맞는 다른 단어 사용
   - ⭕ 제목을 소리 내어 읽었을 때 자연스럽게 들리는지 검증
   - 단, "총정리", "꿀팁", "알아보기", "확인하세요" 같은 낚시성 단어는 여전히 금지

### 예시
- 메인: "바이브코딩"
- 자동완성: 바이브코딩 뜻, 바이브코딩 책, 바이브코딩 앱, 바이브코딩 강의, 클로드 바이브코딩, 바이브코딩 ai
- 제목: "바이브코딩 강의 클로드 바이브코딩 ai" → ❌ (키워드 나열, 딱딱함)
- 제목: "바이브코딩 강의 언제 무료 신청" → ❌ (키워드만 추가한 것, 여전히 딱딱)
- 제목: "바이브코딩 강의 쉬운 ai 앱 만들기" → ⭕ (자연스러운 문장 + "쉬운 ai 앱" 롱테일 가능)
- 제목: "바이브코딩 뜻 요즘 뜨는 ai 코딩" → ⭕ (자연스러움 + "요즘 뜨는 ai 코딩" 롱테일 가능)

## 출력 형식

`blog_titles.json` 파일로 저장 (세션 디렉토리 내):
```json
[
  {
    "keyword": "메인키워드",
    "autocomplete": ["자동완성1", "자동완성2", "..."],
    "selected_keywords": ["선택된키워드1", "선택된키워드2"],
    "title": "생성된 제목",
    "char_count": 15
  }
]
```

## 중요: 점진적 저장 (반드시 준수)

**제목을 1개 생성할 때마다 즉시 blog_titles.json에 저장해야 합니다.**

1. 처음 제목 생성 시: 빈 배열 `[]`로 blog_titles.json 생성
2. 제목 1개를 생성하면 즉시 blog_titles.json을 Read로 읽고, 배열에 새 항목을 추가한 뒤 Write로 덮어쓰기
3. 이 과정을 20개 완료할 때까지 반복
4. **절대 모든 제목을 메모리에 모아놨다가 한 번에 저장하지 말 것** — 중간에 오류가 나면 전부 유실됨

## 작업 흐름

1. 호출자가 전달한 세션 디렉토리 경로 확인
2. 세션 디렉토리의 `autocomplete_data.json` 읽기 → 상위 20개 키워드 추출
3. 세션 디렉토리에 빈 `blog_titles.json` (`[]`) 생성
4. 각 키워드별로:
   a. 규칙에 맞는 제목 1개 생성
   b. 즉시 `blog_titles.json`을 Read → 배열에 추가 → Write로 저장
5. 20개 완료 후 결과 요약 출력
