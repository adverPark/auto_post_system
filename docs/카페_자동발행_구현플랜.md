# 네이버 카페 자동 발행 프로그램 구현 플랜

## Context

애드버코더AI API에서 생성된 카페 포스트(pending 상태)를 Playwright로 네이버 카페에 자동 발행하는 프로그램이 필요하다. 현재 프로젝트는 캠페인 생성 + 폴링까지만 구현되어 있고, 실제 카페 발행(브라우저 자동화)은 미구현 상태.

---

## 파일 구조

| 파일 | 작업 |
|------|------|
| `scripts/publish_cafe.py` | **신규** — 메인 발행 스크립트 (전체 로직) |
| `configs/presets.json` | **수정** — `naver_id`, `naver_pw` 필드 추가 |

- 쿠키 파일: `configs/storage_state_{naver_id}.json` (자동 생성)
- 기존 `scripts/run_campaign.py` 패턴(api_get, headers, argparse, preset 로딩)을 재사용
- `configs/presets.json`이 `.gitignore`에 포함되어 있는지 반드시 확인 (naver_pw 평문 저장)

## 프리셋 확장

```json
{
  "카페프리셋": {
    "description": "카페 자동 발행용",
    "type": "PARS",
    "model": "GEMINI31PRO",
    "publish": "NCF",
    "blog_id": 1,
    "naver_id": "my_naver_id",
    "naver_pw": "my_naver_pw"
  }

```

- `blog_id`: API의 NaverCafe PK (`GET /naver_cafe/cafes/`에서 조회되는 `id`)
- 프리셋별로 다른 네이버 계정/게시판 사용 가능

## API 엔드포인트 정리

> 참고: 사용자 요구사항에서 `GET /naver_cafe/posts/?cafe_id={cafeId}`로 기재되어 있으나, API 문서(`docs/ADVERCODERAI_REST_API.md`)에 따르면 실제 엔드포인트는 아래와 같음

| 용도 | 엔드포인트 | 인증 | 비고 |
|------|-----------|------|------|
| 카페 목록 (api_key 획득) | `GET /naver_cafe/cafes/` | Bearer | cafe_id, menu_id, api_key 반환 |
| pending 포스트 조회 | `GET /naver_cafe/posts/{api_key}/` | 불필요 | pending만 반환 |
| 발행 완료 상태 업데이트 | `POST /naver_cafe/posts/update-status/` | 불필요 | `post_id` + `post_url` (form-urlencoded) |
| 포스트 상태 변경 (실패 시) | `PATCH /naver_cafe/posts/{post_id}/status/` | Bearer | `{"status": "failed"}` |

### 포스트 응답 구조 (9-4 기준 추정, Phase C에서 실제 확인 필요)
```json
{
  "id": 100,
  "status": "pending",
  "cafe_id_str": "mycafe123",
  "menu_id": "2",
  "post": {
    "title": "포스트 제목",
    "post_context": "<h2>소제목</h2><p>본문...</p>"
  }
}
```

## 구현 순서 (3단계)

### Phase A: 로그인 + 쿠키 관리

1. `scripts/publish_cafe.py` 생성 — CLI 스켈레톤 + 프리셋 로딩
2. Playwright async API로 브라우저 실행 (headless=False)
3. **브라우저 context 생성 시 clipboard 권한 부여**:
   ```python
   context = await browser.new_context(
       permissions=["clipboard-read", "clipboard-write"],
       storage_state=storage_path if os.path.exists(storage_path) else None
   )
   ```
4. 로그인 흐름:
   - `configs/storage_state_{naver_id}.json` 존재 → context에 로드 → 로그인 상태 확인
   - 쿠키 없거나 만료 → `nid.naver.com/nidlogin.login`에서 id/pw 로그인
   - 로그인 성공 → storage_state 저장
5. **봇 감지 우회**: `fill()`/`type()` 대신 clipboard paste 방식으로 아이디/비밀번호 입력
   ```python
   await page.evaluate("async (t) => await navigator.clipboard.writeText(t)", text)
   await page.keyboard.press("Meta+KeyV")  # macOS
   ```
6. 로그인 검증: naver.com 접속 후 프로필/로그인 버튼 상태로 판단
7. **Phase A 완료 기준**: 로그인 성공 + 쿠키 저장 확인 + 재실행 시 쿠키로 로그인 스킵 확인. Phase B 진입 전 반드시 완료.

### Phase B: 에디터 동작 확인 (테스트 HTML)

1. `--test-html` 옵션으로 하드코딩된 테스트 HTML로 에디터 검증
2. 카페 글쓰기 URL: `https://cafe.naver.com/ca-fe/cafes/{cafeId}/menus/{menuId}/articles/write`

#### 에디터 구조 (확인됨)

- **제목**: `textarea.textarea_input`

- **소제목 서식 없음** → 인용구로 대체

- [ ] 등록 버튼 클릭 후 페이지 전환 확인 (redirect URL 패턴)

#### HTML → 에디터 태그별 변환 전략

HTML을 통째로 paste하지 않고, **파싱 후 태그별로 에디터 동작을 수행**한다.

`post_context` HTML을 `BeautifulSoup`으로 파싱 → 최상위 요소를 순회하며 각 태그별 처리:

| HTML 태그 | 에디터 변환 | 구현 방법 |
|-----------|-----------|----------|
| `<h2>` | **인용구3** (bubble 스타일, 소제목 역할) | 인용구 툴바 → bubble 선택 → 텍스트 입력 → 인용구 탈출 |
| `<h3>` | **인용구2** (line 스타일) | 인용구 툴바 → line 선택 → 텍스트 입력 → 인용구 탈출 |
| `<p>` | 일반 텍스트 (내부 inline 처리 포함) | 블록 내 children 순회하며 inline 처리 → Enter |
| `<strong>` (블록 레벨) | 볼드 | Cmd+B → 텍스트 입력 → Cmd+B 해제 |
| `<ul>/<li>` | 앞에 "- " 붙인 텍스트 | 각 li마다 `"- {텍스트}"` 입력 → Enter |
| `<ol>/<li>` | 앞에 "1. " 붙인 텍스트 | 각 li마다 `"{번호}. {텍스트}"` 입력 → Enter |
| `<table>` | 클립보드 HTML 붙여넣기 | table HTML → clipboard `text/html` → Cmd+V paste |
| `<p>&nbsp;</p>` | 빈 줄 | Enter 1회 (공백 스페이서 처리) |
| `<img>` | skip (기본) | 이미지는 기본적으로 무시. 필요시 Phase B에서 이미지 삽입 기능 추가 검토 |

#### 액션 모델: 블록 + inline children 구조

`<p>일반 <strong>볼드</strong> 나머지</p>` 같은 mixed content를 처리하기 위해 **2단계 구조**:

```python
def html_to_editor_actions(html: str) -> list[dict]:
    """HTML을 파싱하여 에디터 액션 리스트로 변환.

    블록 레벨 액션 + inline children 구조:
    [
        {"type": "quote_bubble", "children": [{"type": "text", "text": "소제목"}]},
        {"type": "quote_line", "children": [{"type": "text", "text": "부제목"}]},
        {"type": "paragraph", "children": [
            {"type": "text", "text": "일반 "},
            {"type": "bold", "text": "볼드"},
            {"type": "text", "text": " 나머지"}
        ]},
        {"type": "list_item", "children": [{"type": "text", "text": "- 항목"}]},
        {"type": "table_paste", "html": "<table>...</table>"},
        {"type": "empty_line"},
    ]
    """
```

**블록 레벨** (최상위 순회):
- `h2` → `quote_bubble` 블록
- `h3` → `quote_line` 블록
- `p` → `paragraph` 블록
- `ul/ol` → 각 li를 `list_item` 블록으로 변환
- `table` → `table_paste` 블록
- `<p>&nbsp;</p>` → `empty_line` 블록

**inline 레벨** (블록 내부 children):
- 일반 텍스트 → `{"type": "text", "text": "..."}`
- `<strong>` → `{"type": "bold", "text": "..."}`
- `<em>` → `{"type": "italic", "text": "..."}`
- `<a>` → `{"type": "text", "text": "텍스트 (URL)"}`

#### 실행 엔진

```python
async def execute_editor_actions(page, actions: list[dict]):
    """액션 리스트를 순서대로 에디터에 반영."""
    modifier = "Meta" if sys.platform == "darwin" else "Control"
    for action in actions:
        if action["type"] in ("quote_bubble", "quote_line"):
            await apply_quote(page, action["type"], action["children"])
        elif action["type"] == "paragraph":
            await type_inline_children(page, action["children"], modifier)
            await page.keyboard.press("Enter")
        elif action["type"] == "list_item":
            await type_inline_children(page, action["children"], modifier)
            await page.keyboard.press("Enter")
        elif action["type"] == "table_paste":
            await paste_table_html(page, action["html"])
        elif action["type"] == "empty_line":
            await page.keyboard.press("Enter")
        await page.wait_for_timeout(200)

async def type_inline_children(page, children: list[dict], modifier: str):
    """inline children을 순서대로 에디터에 입력."""
    for child in children:
        if child["type"] == "text":
            await page.keyboard.type(child["text"])
        elif child["type"] == "bold":
            await page.keyboard.press(f"{modifier}+KeyB")
            await page.keyboard.type(child["text"])
            await page.keyboard.press(f"{modifier}+KeyB")
        elif child["type"] == "italic":
            await page.keyboard.press(f"{modifier}+KeyI")
            await page.keyboard.type(child["text"])
            await page.keyboard.press(f"{modifier}+KeyI")
```

#### table 처리

```python
async def paste_table_html(page, table_html: str):
    """테이블 HTML을 clipboard text/html로 복사 후 에디터에 paste."""
    await page.evaluate("""async (html) => {
        const blob = new Blob([html], {type: 'text/html'});
        await navigator.clipboard.write([new ClipboardItem({'text/html': blob})]);
    }""", table_html)
    modifier = "Meta" if sys.platform == "darwin" else "Control"
    await page.keyboard.press(f"{modifier}+KeyV")
```

### Phase C: API 연동 + 전체 루프

1. API 함수 구현 (`fetch_cafes`, `fetch_pending_posts`, `update_post_status`, `mark_post_failed`)
2. 메인 루프:
   ```
   카페 목록 조회 (blog_id로 매칭) → api_key, cafe_id, menu_id 획득
   → pending 포스트 조회 (GET /naver_cafe/posts/{api_key}/, 실제 응답 구조 확인)
   → 각 포스트:
     → 글쓰기 페이지 이동
     → textarea.textarea_input에 제목 입력
     → post_context HTML 파싱 → 블록+inline 액션 변환 → 순차 실행
     → a.BaseButton.BaseButton--skinGreen 클릭 (등록)
     → 발행 URL 추출 (page.url)
     → POST /naver_cafe/posts/update-status/ 호출 (성공)
     → 포스트간 딜레이 (기본 5초)
   ```
3. 에러 처리: 포스트 단위 try/except, 실패 시:
   - `PATCH /naver_cafe/posts/{post_id}/status/` → `"failed"` 전송
   - skip → 다음 포스트 진행
4. 결과 요약 출력

## 에러 처리

- **포스트 단위 격리**: 한 포스트 실패 시 → API에 `"failed"` 상태 전송 → skip → 다음 포스트 계속 진행
- **로그인 실패**: headless=False에서 수동 개입 가능 (input() 대기)
- **쿠키 파일 손상**: 삭제 후 재로그인
- **발행 URL 추출 실패**: page.url이 글쓰기 페이지 그대로면 에러 처리
- **실패 포스트 재처리 방지**: failed로 마킹된 포스트는 pending에서 빠지므로 다음 실행 시 재시도 안 함
