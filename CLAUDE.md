# Project Rules

- 가상환경은 uv를 사용할 것.
- 라이브러리 설치는 `uv add`로 할 것!
- 항상 필요한 문서가 있으면 context7을 사용할 것.

# 스킬 실행 규칙

- 스킬(`/command`)이 호출되면 **SKILL.md에 정의된 절차를 반드시 그대로 따를 것**. 절대 자의적으로 다른 방식으로 진행하지 말 것.
- 스킬에서 서브에이전트(Task 도구)에 위임하라고 되어 있으면, 직접 WebSearch/WebFetch 등으로 조사하거나 자체적으로 작업하지 말고 **서브에이전트에 위임할 것**.
- 기존에 만들어진 서브에이전트(AGENT.md)는 수정하지 말고 그대로 사용할 것. 모델도 에이전트에 정의된 것을 따를 것.
- SKILL.md의 `allowed-tools`는 도구를 **자동승인**할 뿐 다른 도구를 **차단하지 않는다**. 도구 제한이 필요하면 AGENT.md의 `tools` 필드를 사용할 것.
- 스킬에서 Task 도구로 서브에이전트를 호출할 때, 모델은 지정하지 말 것. 에이전트 AGENT.md에 정의된 모델이 자동 적용됨.

# 블로그 글 생성 규칙

- 사용자가 요청한 제목은 **절대 변경하지 말 것**. 그대로 사용할 것.
- 메인키워드(`///` 뒤에 붙는 키워드)는 사용자가 따로 요청할 때만 넣을 것. 요청이 없으면 키워드 없이 제목만 전달할 것.

# 주요 워크플로우

- `/full-pipeline <메인키워드> <제목수> <프리셋명>` — 키워드 조사 → 제목 생성 → 캠페인 발행 준비까지 한번에 실행
- `/keyword-collect <키워드>` — 키워드 수집만 단독 실행
- `/keyword-research <키워드>` — 키워드 수집 + 제목 생성 (캠페인 생성 제외)
- `/seo-title-gen` — 이미 수집된 키워드로 제목만 생성
